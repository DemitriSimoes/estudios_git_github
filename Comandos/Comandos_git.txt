
Comandos do Git

git init . -> inicializa o repositorio git vazio (cria uma pasta .git/ no diretorio atual)

git config --global --add safe.directory D:/Analyst/Git_GitHub (caminho para dar permissão de acesso, 
pois o git normalmente verifica quando não tem permissão ou quando está em uma partição diferente, nesse caso D:)

git status -> me mostra como está o meu diretorio

git add .(para pasta atual) ou *(para tudo) ou <nomedo arquivo> -> adiciona os arquivos que eu quero que vá para o commit

git commit -m "<mensagem>" -> cria um save um checkpoint do arquivo, commit feito, alterações salvas

git config --global user.name <nome do usuario> 
git config --global user.email <nome do email> -> adiciona o email que vai usar o git e 
se digitar sem o email só o comando mostra qual email ou usuario que esta usando

git log -> verifica todo o log de commit feito

git diff <nome do arquivo> -> mostra o que foi adicionado e modificado no arquivo

git reset <nome do arquivo> -> tira do git add antes do commit

git checkout -b <nome da branch> -> cria uma nova branch e muda para ela

git checkout <nome da branch> -> muda de branch

git branch -> mostra quais as branchs que existem e com * mostra qual está

git merge <nome da branch> -> tras as alterações feitas na outra branch, ou seja, tras o que foi feito commit
** importante: se der erro no momento do merge, entrar no arquivo olhar as alterações, confirmar ou não,
 salvar o arquivo, dar git add <nome do arquivo>, depois git commit (vai abrir o Vim, ai apertar esc :wq)
 
git stash -> para suspender as mudanças e poder mudar de branch

git stash pop -> quando voltar para a branch que deu stash o pop recupera o que foi feito pode fazer o git add e git commit

git branch -D <nome da branch> -> apaga a branch, mas não pode estar dentro dela

git remote add origin <codigo SSH> ou <codigo html> -> faz a ligação entre o repositorio do GitHub com o versionamento do Git local

git push origin main -> envia o ultimo commit para o repositorio GitHub

git pull origin main -> atualiza minha base local do main com o ultimo commit do repositorio GitHub

.gitignore -> arquivo criado com esse nome para excluir arquivo que não fazem parte dos commits<br>
normalmente arquivos que são binários e não de texto

.gitkeep -> arquivo criado dentro de uma pasta para que a pasta não seja ignorada caso esteja vazia

Fluxo de trabalho
git init .
git status
git add . ou nome do arquivo
git commit -m "<mensagem>"
git status
git log
